### 🟡 P1 - 产品资料（推荐用Admin后台）

**方案A：通过Admin后台修改（最安全，不用重新部署）**
1. 访问 `https://你的域名/admin`
2. 登录密码：默认 `admin123`（记得改！）
3. 直接编辑产品：名称、描述、规格、价格提示
4. 实时生效，无需部署

**方案B：直接编辑数据库（最快）**
```bash
cd /path/to/plastic-export
npx prisma studio
# 打开可视化编辑器，直接修改 Product 表
```

**产品需要补充的字段**：
- `name` / `nameEn` - 中英文产品名
- `description` - 详细描述（不只是"通用级"，要写清楚卖点）
- `specs` - 技术规格（密度、熔指、拉伸强度等）
- `priceHint` - 价格区间（FOB Shanghai $xxx/MT）
- `baseMaterial` - 基材类型（PA66, PC, PP等）
- `applicationTags` - 应用场景标签（JSON格式：`["automotive","electrical"]`）
- `image` - 图片路径

### 🟡 P1 - 图片修复

**上传路径**：
```
/plastic-export/public/images/products/
```

**数据库更新**：
在Admin后台或数据库中设置 `image` 字段：
```
/images/products/你的图片文件名.jpg
```

**图片要求**：
- 格式：JPG/PNG
- 尺寸：建议 800x800 或 1200x1200
- 大小：每张 < 500KB（网页加载快）
- 命名：用英文，无空格（如：`pa66-gf30-granules.jpg`）

## 🚀 部署流程

### 如果你用Docker部署（推荐）

```bash
# 1. 本地修改代码后，构建镜像
docker build -t plastic-export:latest .

# 2. 保存镜像
docker save plastic-export:latest > plastic-export.tar

# 3. 传到服务器（scp或rsync）
scp plastic-export.tar user@你的服务器:/tmp/

# 4. 服务器上加载并重启
ssh user@你的服务器
docker load < /tmp/plastic-export.tar
cd /path/to/docker-compose
docker-compose down && docker-compose up -d
```

### 如果你用Git部署

```bash
# 服务器上执行
cd /path/to/plastic-export
git pull                    # 拉取最新代码
npm install                 # 安装依赖
npx prisma migrate deploy   # 数据库迁移
npm run build              # 构建
pm2 restart plastic-export # 重启服务
```
